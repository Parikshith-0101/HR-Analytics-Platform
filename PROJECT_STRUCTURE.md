## 📊 File Organization by Purpose

### 1. **Datasets** (`datasets/`)
- **Purpose**: Store raw input data
- **Contains**: Original CSV files downloaded from data sources
- **Access**: Read-only for exploratory analysis and preprocessing

### 2. **Processed Data** (`processed_data/`)
- **Purpose**: Store preprocessed datasets ready for model training
- **Generated by**: `preprocessing.py`
- **Used by**: `model_selection.py`, `model_training_and_tuning.py`, `threshold_optimization.py`
- **Files**:
  - Split datasets (X_train, X_test, y_train, y_test)
  - Feature-scaled training/test data
  - Full processed dataset with selected features

### 3. **Preprocessing Artifacts** (`preprocessing_artifacts/`)
- **Purpose**: Store serialized preprocessing objects needed for production/inference
- **Generated by**: `preprocessing.py`
- **Used by**: Production pipelines for applying same transformations
- **Files**:
  - Scaler (StandardScaler) for feature normalization
  - Label encoders (for categorical variables)
  - Feature list (selected features used in training)

### 4. **Models** (`models/`)
- **Purpose**: Store trained machine learning models
- **Generated by**: `model_selection.py` and `model_training_and_tuning.py`
- **Used by**: `threshold_optimization.py` and production pipelines
- **Files**:
  - `best_attrition_model.pkl` - Best performing model from comparison
  - `tuned_attrition_model.pkl` - Hyperparameter-tuned model

### 5. **Results** (`results/`)
- **Purpose**: Store model evaluation results and metrics
- **Generated by**: `model_selection.py`, `model_training_and_tuning.py`, `threshold_optimization.py`
- **Files**:
  - Model comparison metrics (Accuracy, Precision, Recall, F1, ROC-AUC)
  - Best hyperparameters from grid search
  - Threshold optimization results

### 6. **Visualizations** (`visualizations/`)
- **Purpose**: Store generated plots and charts
- **Contains**: Confusion matrices, performance plots, etc.

## 🔄 Pipeline Flow

```
1. EDA.ipynb
   ↓ (reads from datasets/)
   
2. preprocessing.py
   ├─ Input: datasets/WA_Fn-UseC_-HR-Employee-Attrition.csv
   ├─ Output: 
   │  ├─ processed_data/*.csv
   │  └─ preprocessing_artifacts/*.pkl
   
3. model_selection.py
   ├─ Input: processed_data/*
   ├─ Output:
   │  ├─ models/best_attrition_model.pkl
   │  └─ results/model_comparison_results.csv
   
4. model_training_and_tuning.py
   ├─ Input: processed_data/*
   ├─ Output:
   │  ├─ models/tuned_attrition_model.pkl
   │  └─ results/best_hyperparameters.csv
   
5. threshold_optimization.py
   ├─ Input: 
   │  ├─ processed_data/* (test data)
   │  └─ models/tuned_attrition_model.pkl
   └─ Output:
      ├─ results/final_threshold.txt
      └─ results/final_threshold_results.csv

6. final_evaluation.py [EVALUATION STAGE]
   ├─ Input:
   │  ├─ processed_data/* (test data)
   │  ├─ models/tuned_attrition_model.pkl
   │  └─ results/final_threshold.txt
   └─ Output:
      ├─ results/final_evaluation_metrics.png
      ├─ results/probability_distribution.png
      ├─ results/final_evaluation_report.txt
      ├─ results/final_metrics_summary.csv
      └─ results/test_predictions_detailed.csv

7. prediction_pipeline.py [PRODUCTION STAGE]
   ├─ Input:
   │  ├─ models/tuned_attrition_model.pkl
   │  ├─ preprocessing_artifacts/* (all encoders & scaler)
   │  ├─ results/final_threshold.txt
   │  └─ New employee data (CSV or manual entry)
   └─ Output:
      └─ Predictions with probabilities & risk levels
      └─ results/predictions_YYYYMMDD_HHMMSS.csv
```

## � Pipeline Stages

### Stage 1: Data Preparation (EDA → Preprocessing)
- **EDA.ipynb**: Exploratory analysis and feature correlation
- **preprocessing.py**: Feature selection, scaling, encoding, and train-test split

### Stage 2: Model Development (Selection → Tuning)
- **model_selection.py**: Compare 4 different ML algorithms
- **model_training_and_tuning.py**: Hyperparameter optimization using GridSearchCV

### Stage 3: Threshold Optimization (Optimization)
- **threshold_optimization.py**: Find optimal decision threshold balancing business metrics

### Stage 4: Final Evaluation (Evaluation)
- **final_evaluation.py**: Comprehensive test set evaluation with visualizations and reports
- **Output**: Metrics, visualizations, and detailed performance analysis

### Stage 5: Production Deployment (Prediction)
- **prediction_pipeline.py**: Production-ready prediction system
- **Modes**: Batch CSV, manual entry, or demo predictions
- **Output**: Risk stratification and business recommendations
